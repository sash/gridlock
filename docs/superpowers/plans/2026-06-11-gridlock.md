# GridLock Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the complete GridLock block-placement puzzle from `specs.md` — all versions v1→v1.3 (Classic, Daily, Rush, Zen; specials; power-ups; juice; PWA) — with unit tests and a browser e2e test.

**Architecture:** Pure, serializable game core (`src/core`, zero DOM/Pixi deps, fully unit-tested with Vitest) driven by a PixiJS 8 view layer (`src/ui`) that renders board/tray, handles pointer drag with lift offset + ghost preview, and plays synthesized WebAudio sounds. State persists to localStorage on every placement. Playwright e2e drives the real browser via pointer events and a `window.__game` debug handle.

**Tech Stack:** TypeScript, Vite, PixiJS 8, Vitest, Playwright (chromium), WebAudio (synthesized — no audio assets), PWA manifest + service worker.

## Spec interpretation decisions (ambiguities resolved)

- **Cell encoding** (`Uint8Array(64)`): `0` empty; `1..8` filled (value = color index); `16` gem, `17` ice, `18` cracked ice, `19` bomb, `20` stone, `21` wild. All values ≥1 count as "filled" for placement blocking and line-completion checks.
- **Gem** spawns on an empty cell and *occupies* it (filled blocker + reward). Clearing its line: cell empties, +150.
- **Ice** converts a filled cell. First clear → becomes `cracked` (stays filled); second clear → empties.
- **Bomb** spawns on a random empty cell (occupies it), counter starts at 9 and decrements per placement. Line-cleared in time → empties a 3×3 area centered on it (bonus, no extra line points). Counter 0 → becomes `stone`.
- **Stone** counts as filled for line checks but *survives* clears; expires to empty 15 placements after petrifying.
- **Wild** (reward for 3+ line clear) spawns on a random empty cell, occupies it, counts as filled for every row/column check, clears normally. Pure freebie.
- **Aux state** (bomb counters, stone timers) lives beside the board: `{ bombs: Record<index, count>, stones: Record<index, count> }` — serialized with the rest.
- **Undo** keeps one snapshot of the pre-placement state; unavailable if the last placement caused a clear, after use, or before any placement.
- **Power-up earning:** +1 of each on first game of a calendar day (daily login); +1 random power-up when streak hits cap (×5); +1 random on perfect clear. Inventory persisted. Max 1 *use* of each per Classic game.
- **Zen dissolve:** when no tray piece fits, the 2 rows with the most filled cells (ties → lower row index first) empty; no points.
- **Rush:** 90s; each placement immediately deals a replacement piece into that tray slot; game over on timeout (core exposes `timeLeft` driven by the UI ticker).
- **Daily:** seed = `yyyymmdd` int; ~10 prefilled cells from seeded RNG; one attempt/day (localStorage); share card = Wordle-style emoji grid of final board + score.

## File structure

```
src/core/rng.ts          mulberry32 seeded RNG, serializable state
src/core/pieces.ts       shape table (14 logical shapes → ~30 dealt variants), rotation map, weights
src/core/board.ts        Uint8Array board ops: canPlace, place, findCompletedLines, applyClears, anyFit
src/core/scoring.ts      line points table, streak multiplier, perfect-clear bonus
src/core/generator.ts    weighted bag + solvability check + <40% guarantee + pity rule
src/core/specials.ts     spawn rules + per-clear effects for gem/ice/bomb/stone/wild
src/core/game.ts         Game class: full state machine, modes, power-ups, undo, (de)serialize
src/core/daily.ts        date→seed, prefill, share-card emoji grid
src/ui/*                 Pixi app, board/tray views, drag controller, effects, HUD, menu, audio, storage
tests/*.test.ts          Vitest unit tests per core module
e2e/game.spec.ts         Playwright browser test
public/manifest.webmanifest, public/sw.js, index.html
```
